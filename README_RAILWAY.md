# BTC Wallet Bot — same-word generators + PIN history

Railway-версия Telegram-бота для генерации Bitcoin P2PKH кошельков.

## Что добавлено

- Обычные кнопки генерации: `🎲 12 слов`, `🎲 24 слова`.
- Новые кнопки:
  - `🎯 Рандом12 одинаковые` — бот выбирает 1 случайное слово из BIP39-списка 2048 слов и делает фразу из 12 одинаковых слов.
  - `🎯 Рандом24 одинаковые` — аналогично, но 24 одинаковых слова.
- История с PIN:
  - `🔐 Установить PIN` или команда `/set_pin 12345`.
  - PIN должен быть ровно 5 цифр.
  - `📜 История` просит PIN один раз за сессию бота; после перезапуска PIN потребуется снова.
  - После установки PIN новые записи истории сохраняют ключи за PIN; TXT-экспорт выдаёт два файла: `public.txt` и `private.txt`.
- Баланс проверяется через Blockstream, mempool.space и BlockCypher.

## Важно по безопасности

Фразы из одинаковых слов крайне небезопасны. Не переводите реальные деньги на такие кошельки.

PIN из 5 цифр защищает просмотр истории в Telegram. Секреты в файле истории шифруются Fernet-ключом, который хранится в `history_secret.key` или может быть задан через Railway Variable `HISTORY_SECRET_KEY`.

Если потерять `history_secret.key` или `HISTORY_SECRET_KEY`, старые сохранённые WIF/mnemonic из истории расшифровать не получится.

## Railway Variables

Минимально:

```env
TELEGRAM_BOT_TOKEN=123456:ABCDEF
```

Опционально:

```env
DATA_DIR=/data
HISTORY_SECRET_KEY=base64_fernet_key
```

Для постоянного хранения подключи Railway Volume и mount path `/data`.

Railway использует:


## Пакетный режим

В этой версии добавлена кнопка `⚙️ Пакет: ВКЛ/ВЫКЛ`.

- Когда режим выключен, кнопки `🎲 12 слов`, `🎲 24 слова`, `🎯 Рандом12 одинаковые` и `🎯 Рандом24 одинаковые` создают по одному кошельку, как раньше.
- Когда режим включён, кнопки `🎲 12 слов`, `🎲 24 слова`, `🎯 Рандом12 одинаковые` и `🎯 Рандом24 одинаковые` создают сразу 10000 новых кошельков.
- Для пакетного режима нужен PIN: приватные ключи сохраняются в зашифрованную историю и открываются через `📜 История`.
- В пакетном режиме баланс не проверяется во время генерации. Когда создаётся `public_*.txt` через `📋 Копировать всё`, бот сразу запускает проверку баланса по публичным адресам из этого TXT.
- Кнопка `🔑 Приват ключ: ВКЛ/ВЫКЛ` включает режим, в котором генерация создаёт random private key → WIF без вывода seed-фраз.
- Настройки режима хранятся в `wallets_settings.json` рядом с историей.

## TXT-экспорт последних 10000 ключей

Кнопка `📋 Копировать всё` запрашивает PIN один раз за сессию и отправляет два `.txt` файла по последним 10000 сохранённым записям:

- `public_*.txt` — публичные BTC-адреса, один адрес в строке;
- `private_*.txt` — приватные ключи WIF, один ключ в строке.

Номера строк в двух файлах совпадают: строка 1 в `public_*.txt` соответствует строке 1 в `private_*.txt`.

Seed-фразы, баланс, checksum, тип, дата и derivation path из TXT-экспорта убраны.

После отправки `public_*.txt` и `private_*.txt` бот автоматически запускает пакетную проверку баланса по адресам из созданного `public_*.txt`. `private_*.txt`, WIF и seed при этой проверке не читаются и не обрабатываются. Счётчик `📊 Проверено` увеличивается во время проверки и хранится только до перезапуска бота.

⚠️ WIF даёт полный доступ к кошельку. Храните `private_*.txt` офлайн и никому не отправляйте.

## Проверка баланса по загруженному TXT/CSV

Можно загрузить `public_*.txt` или любой TXT/CSV со списком публичных BTC-адресов — по одному адресу в строке. Бот проверит до 10000 адресов и добавит адреса с балансом `> 0` в RAM-кнопку `💰 Положительный баланс`. Положительные адреса также сохраняются в историю без приватных ключей.

Ограничения:

- файлы с seed/WIF/private-key данными не принимаются;
- проверяются только публичные адреса;
- найденные положительные адреса сохраняются в историю только как `address + balance`, без приватных ключей;
- RAM-список положительных балансов живёт только до перезапуска бота; история адресов сохраняется.


## Восстановительный скан seed

Команда `/scan10000 seed words...` проверяет первые 10000 адресов из введённой seed-фразы по пути `m/44'/0'/0'/0/i`. Старые алиасы `/scan1000` и `/scan100` оставлены для совместимости.


## Ускоренная проверка public.txt

Для максимальной скорости проверки публичных адресов можно добавить в Railway Variables:

```env
BALANCE_BATCH_SIZE=100
BALANCE_BATCH_WORKERS=16
BALANCE_SCAN_WORKERS=48
BALANCE_REQUEST_TIMEOUT=5
BLOCKCYPHER_TOKEN=
```

`BLOCKCYPHER_TOKEN` необязателен, но с ним меньше риск лимитов API. Проверка работает только для TXT/CSV с публичными адресами или HEX public-key; WIF/seed/private-key блокируются.
